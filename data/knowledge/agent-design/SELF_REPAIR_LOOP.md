# Self Repair Loop

The self-repair loop is:

- perceive
- understand
- plan
- act
- verify
- remember
- improve

Perceive means inspect the current environment honestly. Understand means classify the task, risks, and available authority. Plan means choose the smallest useful next action. Act means make the approved change. Verify means prove the result with checks. Remember means save receipts and useful lessons. Improve means use the proof to make the next loop better.

This first X Native baseline only implements perceive, remember, and proof readiness. It can receive prompt packages, inspect repository readiness, and save receipts, but it does not yet apply arbitrary patches or execute autonomous system actions.
